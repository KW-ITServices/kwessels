Azure Troubleshooting Cue Cards — 14 Common Issues

Compact study guide with quick steps, ITIL & Agile notes, and a printable cram sheet.

1. What is Azure Active Directory, and how does it differ from on-premises AD?
  1. Azure AD is a cloud-based identity and access management service.
  2. On-prem AD manages devices and users locally using domain join, GPO, Kerberos.
  3. Azure AD supports SSO, MFA, Conditional Access, identity governance.
ITIL: Identity impacts availability & access Agile: iterative policy rollouts
2. How do you troubleshoot login issues in Azure AD?
  1. Check user state: active, not blocked, password validity.
  2. Use sign-in logs to identify authentication or MFA failures.
  3. Review Conditional Access policy assignments.
  4. Validate device compliance in Intune.
  5. Check Azure AD Connect for sync errors.
  6. Document and communicate resolution.
3. Explain Azure Resource Groups and why they are important
  1. Logical containers for Azure resources.
  2. Enable RBAC, lifecycle control, cost management.
  3. Support tagging, automation, structured deployments.
4. How do you monitor Azure resources effectively?
  1. Azure Monitor dashboards and alerts.
  2. Log Analytics for logs & trend analysis.
  3. Application Insights for app telemetry.
  4. Network Watcher for flow & connectivity monitoring.
  5. Integrate alerts with ITSM for automated incidents.
5. Troubleshooting a failing Azure VM
  1. Check VM state (Running/Stopped).
  2. Check Boot Diagnostics for errors.
  3. Review CPU, RAM, IOPS metrics.
  4. Check NSG/firewall/IP config.
  5. RDP/SSH and inspect OS logs.
  6. Redeploy or restore from snapshot if needed.
6. Implement Azure RBAC
  1. Identify least-privilege role needed.
  2. Assign at correct scope (sub/resource group/resource).
  3. Use groups, avoid individual assignments.
  4. Test permissions.
  5. Review regularly for stale permissions.
7. Troubleshoot Azure Storage issues
  1. Check storage health and service status.
  2. Validate firewall/VNet/private endpoints.
  3. Verify keys/SAS/RBAC permissions.
  4. Check for throttling/quota limits.
  5. Test across clients & review logs.
8. Troubleshoot Azure Backup failures
  1. Check Recovery Services Vault job status.
  2. Review backup logs and error codes.
  3. Validate storage and network connectivity.
  4. Test restore to confirm integrity.
  5. Adjust retention or schedule if needed.
9. Troubleshoot Azure Networking
  1. Check IP, VNet, NSG, UDR configuration.
  2. Use Network Watcher tools.
  3. Ping/tracert + flow logs.
  4. Fix misconfigured rules or routes.
10. Troubleshoot hybrid Azure AD sync issues
  1. Check Azure AD Connect health & sync service.
  2. Identify missing/duplicate attributes.
  3. Fix AD objects or filtering.
  4. Re-run sync and confirm.
11. Troubleshoot Conditional Access / MFA issues
  1. Identify impacted apps/users.
  2. Review sign-in logs for policy blocks.
  3. Verify device compliance.
  4. Reset MFA or adjust policy.
12. Troubleshoot VM performance issues
  1. Review CPU/RAM/IOPS metrics.
  2. Inspect OS resource usage.
  3. Optimize apps & disk.
  4. Scale VM size if necessary.
13. Troubleshoot Azure SQL connectivity
  1. Check DB/server status.
  2. Validate firewall/VNet rules.
  3. Test from SSMS/Azure Data Studio.
  4. Fix routing/IP/credentials issues.
14. Prevent Azure outages proactively
  1. Build dashboards & automated alerting.
  2. Implement redundancy & geo-replication.
  3. Review RBAC and policies regularly.
  4. Test backups + DR plans.
  5. Perform post-mortems and improve runbooks.

  • Storage: Health → Permissions → Throttle.
  • Backup: Vault → Logs → Restore test.
  • Network: NSG → UDR → Watcher.
