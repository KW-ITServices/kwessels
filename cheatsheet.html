Azure Troubleshooting — 14 Common Issues

1. What is Azure Active Directory, and how does it differ from on-premises AD?
  1. Azure AD is a cloud-based identity & access management service for Microsoft cloud services and SaaS.
  2. On-prem AD controls domain join, Group Policy, Kerberos authentication for local networks.
  3. Azure AD provides SSO, MFA, Conditional Access, identity protection and OAuth/OpenID Connect for apps.
ITIL: identity affects access & availabilityAgile: iterative policy rollouts
2. How do you troubleshoot login issues in Azure AD?
  1. Verify user state (active, not blocked) and password validity.
  2. Check Azure AD sign-in logs for error codes (MFA fail, CA block, impossible travel).
  3. Review Conditional Access policy assignments and grant controls.
  4. Confirm device compliance via Intune and device registration status.
  5. Inspect Azure AD Connect for sync failures (synchronization service manager).
  6. Document root cause and update KB for recurring scenarios.
3. Explain Azure Resource Groups and why they are important
  1. Logical containers that group related Azure resources (VMs, storage, DBs) for lifecycle management.
  2. Enable RBAC, tagging, cost tracking, and easier automation/deployment.
  3. Support deletion, movement and monitoring as a unit for operational control.
4. How do you monitor Azure resources effectively?
  1. Use Azure Monitor for metrics, alerts and dashboards.
  2. Use Log Analytics to query logs and detect trends.
  3. Use Application Insights for app telemetry and failures.
  4. Use Network Watcher for connectivity and VM-level packet captures.
  5. Integrate with ITSM for automated incident creation (ServiceNow/Jira).
5. How would you troubleshoot a failing Azure Virtual Machine?
  1. Check VM state in the Portal (Running/Stopped/Deallocated).
  2. Review Boot Diagnostics (serial log, screenshot) for boot or OS errors.
  3. Check Azure Monitor metrics: CPU, memory, disk IOPS and network throughput.
  4. Inspect NSGs, NIC settings, private/public IP and LB health probes.
  5. RDP/SSH into the VM to check OS event logs and services.
  6. Redeploy VM, swap host, or restore from snapshot if corruption suspected.
6. How do you implement Azure Role-Based Access Control (RBAC)?
  1. Identify least-privilege roles required (Owner, Contributor, Reader or custom).
  2. Assign at appropriate scope (subscription, resource group, resource).
  3. Use Azure AD groups for assignments to simplify management.
  4. Test using a non-privileged account and schedule periodic reviews.
  5. Document assignments for audit and compliance purposes.
7. How do you troubleshoot Azure Storage issues (Blob, File, Disk)?
  1. Check Storage Account health and Azure service status.
  2. Validate network connectivity, firewall and endpoint configurations.
  3. Check authentication: keys, SAS tokens expiry, RBAC roles.
  4. Look for throttling, quota limits or transaction spikes in metrics.
  5. Test from alternate endpoints and review diagnostic logs.
8. Troubleshoot Azure Backup failures
  1. Open Recovery Services Vault and check job status and error codes.
  2. Review logs for reasons (file locks, network, authentication).
  3. Verify target storage availability and connectivity.
  4. Test restores to a non-prod VM to validate backups.
  5. Adjust schedules/retention and document changes.
9. How do you troubleshoot Azure Networking issues?
  1. Validate VNet, NSG, UDR, and subnet configs.
  2. Use Network Watcher: connection troubleshoot, IP flow verify, packet capture.
  3. Run tracert/ping to isolate routes and latency.
  4. Verify Load Balancer health probes and firewall rules.
10. How would you handle a hybrid Azure AD sync issue?
  1. Inspect Azure AD Connect Health & Synchronization Service Manager for errors.
  2. Identify missing attributes, duplicates, or attribute mapping issues.
  3. Fix on-prem AD objects or OU filtering and re-run delta/full sync.
  4. Validate users can authenticate and document the fix.
11. Troubleshoot Conditional Access or MFA issues
  1. Identify affected users and impacted applications.
  2. Check sign-in logs for blocked attempts and policy triggers.
  3. Review device compliance and grant controls in CA policies.
  4. Reset MFA for affected users as a remediation step if appropriate.
  5. Document policy changes and test in a pilot group prior to broad rollout.
12. How do you handle Azure VM performance issues?
  1. Review Azure Monitor metrics: CPU, memory, disk IOPS, network throughput.
  2. Inspect guest OS processes and resource usage.
  3. Identify resource-hungry applications and optimize them.
  4. Evaluate resizing VM or using scale sets and optimize storage types.
  5. Monitor after fix and document the performance plan.
13. How to troubleshoot Azure SQL Database connectivity issues
  1. Confirm server and database status in the Portal.
  2. Validate firewall rules, VNet service endpoints or private endpoints.
  3. Test connectivity with SSMS or Azure Data Studio and check connection strings.
  4. Inspect error codes and verify credentials and permissions.
  5. Apply fixes (whitelisting IPs, scaling DB, restarting services) and document.
14. How do you proactively prevent Azure service outages?
  1. Implement monitoring dashboards, alerts, and automated runbooks.
  2. Apply redundancy and geo-replication for critical resources.
  3. Review and test RBAC, Conditional Access and other policies regularly.
  4. Regularly test backups, DR failover and restore procedures.
  5. Perform post-incident reviews and feed learnings into automation and runbooks.
ITIL: continual improvementAgile: iterative testing

RMM & Automation

RMM 1. How do you manage, tune, and optimize RMM alerts to reduce noise?
  1. Classify alerts by impact and severity; suppress low-value ones.
  2. Tune thresholds using historical baselines to minimize false positives.
  3. Create alert dependencies and maintenance windows to avoid duplicate alerts.
  4. Automate remediation (service restart, log rotate, disk cleanup) where safe.
  5. Audit alert rules routinely and track alert-to-ticket ratios.
Goal: signal > noiseKPI: reduce alert fatigue
RMM 2. Patch management process across clients
  1. Segment endpoints by OS, role and criticality.
  2. Test patches in pilot groups then stagger full deployment.
  3. Automate reboots and verification using the RMM's task engine.
  4. Document exceptions and follow ITIL change process for risky patches.
  5. Produce compliance reports for clients.
RMM 3. Proactive monitoring strategy
  1. Baseline monitors for CPU, memory, disk, services, event logs and backups.
  2. Custom monitors for business apps and integrations.
  3. Automated ticket creation & remediation runbooks for common events.
  4. Scheduled health reports and trend analytics to find weak spots.
RMM 4. Examples of automation scripts used in an MSP
  1. Auto-restart critical services when stopped and log the event.
  2. Disk cleanup and log rotation scripts to prevent disk exhaustion.
  3. Automated certificate expiry checks and renew notifications.
  4. Agent health remediation (reinstall or redeploy agent) via RMM policy.

Microsoft 365

M365 1. Exchange Online mail flow troubleshooting
  1. Run Message Trace to find delivery status and error codes.
  2. Confirm SPF, DKIM and DMARC DNS records and alignment.
  3. Check transport rules, spam policies and quarantines.
  4. Review connectors in hybrid setups and check TLS cert validity.
M365 2. Teams sign-in & connectivity troubleshooting
  1. Check sign-in logs for CA/MFA blocks and token errors.
  2. Clear Teams cache or reinstall app; test web client.
  3. Ensure network allows Teams endpoints and media ports.
  4. Verify service health on Microsoft 365 admin center.
M365 3. SharePoint / OneDrive sync & permission issues
  1. Check site collection and file-level permissions; inheritance breaks cause errors.
  2. Validate OneDrive sync client logs and version.
  3. Inspect tenant-level DLP or sensitivity labels blocking sync.
  4. For access issues, ensure correct tenant login and licensing.
M365 4. Intune enrollment & compliance troubleshooting
  1. Verify licensing and MDM authority; Intune + Azure AD P1 required for advanced features.
  2. Check enrollment restrictions and device limits per user.
  3. Confirm Autopilot profiles and MDM policies apply correctly.
  4. Inspect device logs for enrollment errors and sync status.

Windows Server / Active Directory

AD 1. Domain login failures — root cause checklist
  1. Check account state (locked, expired, disabled) and last password change.
  2. Ensure DNS resolves DCs (SRV records) and clients can reach DCs over LDAP/SMB.
  3. Check time sync — Kerberos authentication fails with >5 minute drift.
  4. Look for Kerberos errors, replicate and check DC health.
AD 2. GPO not applying — troubleshooting steps
  1. Run gpresult /r and gpupdate /force.
  2. Check link order, inheritance, security filtering and WMI filters.
  3. Verify SYSVOL and NETLOGON shares and DFSR/replication health.
  4. Look for permission issues on GPO objects or missing templates.
AD 3. NTFS vs Share permissions & file-lock troubleshooting
  1. Remember effective permission = most restrictive (explicit DENY overrides).
  2. Audit group membership and nested groups causing unexpected access.
  3. Check open sessions and file locks on file server tools.
  4. Investigate DFS namespace issues and client offline files caching.
AD 4. Advanced AD topics: FSMO roles & replication issues
  1. Know FSMO roles: Schema, Domain Naming, PDC Emulator, RID Master, Infrastructure.
  2. Use repadmin /replsummary and dcdiag to check health.
  3. For failed DCs, consider metadata cleanup and seizure of FSMO roles only if permanent failure.
  4. Watch for USN rollback/tombstone lifetime issues after restoring old snapshots.

Networking — Basic to Advanced

NET 1. IP Addressing, subnetting & conflict diagnosis
  1. IPv4 (32-bit) and IPv6 (128-bit) fundamentals; CIDR notation defines subnet masks.
  2. To diagnose conflicts: check DHCP leases, run arp -a, scan network for duplicate MACs, isolate static devices.
  3. Use DHCP reservations for critical devices to avoid conflicts.
NET 2. DNS troubleshooting essentials
  1. Verify client DNS settings (NIC/DHCP scope) and test resolution with nslookup.
  2. Check DNS records (A, CNAME, MX, SRV) and replication for AD-integrated zones.
  3. Flush caches (ipconfig /flushdns) and check split-brain DNS if hybrid.
NET 3. DHCP issues & relay troubleshooting
  1. Check scope exhaustion and available leases.
  2. Verify DHCP service authorized (Windows) and scope options.
  3. Confirm IP helper / DHCP relay on L3 devices for multi-VLAN environments.
NET 4. VLANs, trunking & troubleshooting native VLAN issues
  1. VLANs logically separate traffic; trunks carry multiple VLANs with 802.1Q tagging.
  2. Troubleshoot with show interfaces trunk, check allowed VLAN lists and native VLAN mismatches.
  3. Ensure DHCP and routing for VLANs are configured on L3 device or firewall.
NET 5. VPN troubleshooting: IPsec & SSL VPNs
  1. Identify VPN type (IPsec IKEv1/IKEv2, SSL, OpenVPN) and check phase 1/phase 2 negotiations.
  2. Verify shared secrets, certificate validity, and proposals (encryption/auth algorithms).
  3. Check firewall ports (UDP 500/4500, ESP) and path MTU/fragmentation issues.
NET 6. Firewall policy troubleshooting & NAT
  1. Review firewall logs for deny hits and compare rule ordering.
  2. Check NAT rules for correct translation; ensure inbound mapping and PAT setup.
  3. Watch for rule shadowing and implicit deny rules at the end of policy.
NET 7. Wi-Fi troubleshooting: RSSI, interference & roaming
  1. Check AP signal (RSSI), channel overlap and co-channel interference.
  2. Use spectrum analysis to find non-Wi-Fi interference (microwaves, Bluetooth).
  3. Tune band steering, roaming thresholds and minimum RSSI to avoid sticky clients.
  4. Ensure CAPWAP/Controller connectivity and correct VLAN tagging on SSIDs.
NET 8. Advanced routing: OSPF vs BGP & troubleshooting
  1. OSPF: IGP for internal routing; check adjacency states and LSDB.
  2. BGP: used for multi-WAN and ISP peering; inspect AS path, local-pref, MED and communities.
  3. Troubleshoot neighbors, authentication, and route propagation; check for route flapping or next-hop issues.
NET 9. Packet loss & latency diagnosis
  1. Start local → gateway → ISP pings and run traceroute/pathping to isolate segments.
  2. Check interface errors, duplex mismatch, CRC and overruns on switches/routers.
  3. Investigate CPU/memory spikes on firewalls or routers causing drops.
NET 10. HA, failover & link aggregation (LACP)
  1. HA: verify heartbeat links, config sync and matching firmware/licensing.
  2. Test failover procedures in maintenance windows and monitor for split-brain.
  3. LACP: ensure consistent LACP mode and hashing across both ends and check LACP negotiation.

Infrastructure & Servers

INF 1. Virtualization – Hyper-V & VMware troubleshooting
  1. Check hypervisor host health (CPU, memory, storage latency) and licensing.
  2. Inspect VM tools/agents and ballooning or memory reservations.
  3. For storage issues, check datastore latency, path redundancy and multipathing.
  4. Use snapshot/backup best practices and avoid long-lived snapshots.
INF 2. Backup & DR – strategy and troubleshooting
  1. Verify backup jobs, retention, and RPO/RTO against SLAs.
  2. Test restores regularly to validate data integrity.
  3. Document runbooks for restore and failover; interview stakeholders about critical systems.
  4. For cloud backups, check API throttling and service limits that may cause failures.
INF 3. Storage troubleshooting (SAN/NAS) & performance
  1. Monitor storage latency, IOPS and throughput; correlate with VM/host activity.
  2. Check multipathing, firmware/driver mismatches and FC/iSCSI connectivity.
  3. Review deduplication, thin provisioning and thin-clone behavior for capacity surprises.

Security & Identity

SEC 1. Responding to suspicious login / account compromise
  1. Collect sign-in evidence: locations, devices, IPs, and impossible travel alerts.
  2. Block account, force password reset, revoke sessions and refresh tokens.
  3. Scan mailbox for suspicious forwarding rules and search for lateral movement indicators.
  4. Elevate to security team & run forensic captures if necessary.
SEC 2. Conditional Access policies blocking legitimate users
  1. Identify sign-in log entries and which policy evaluated to block the login.
  2. Create a temporary exclusion group for troubleshooting and re-enable after fix.
  3. Simplify policies: split them by app/user group rather than huge catch-all rules.
  4. Document and test in pilot groups before full rollout.

Scenario-Based Questions

Scenario 1. SEV1 – Microsoft 365 outage
  1. Confirm scope via Microsoft 365 Service Health and known outages.
  2. Communicate immediate impact and next steps to stakeholders.
  3. Implement workarounds (cached Outlook, mobile apps, alternate comms).
  4. Track status and prepare post-incident review focusing on mitigation & SLAs.
Scenario 2. On-prem AD domain controller failure
  1. Check DC services (NTDS, DNS, Netlogon) and available SYSVOL share.
  2. Use repadmin /replsummary and dcdiag to assess replication issues.
  3. If permanent failure, perform metadata cleanup and seize FSMO roles carefully.
  4. Document the incident and re-evaluate DC redundancy design.
Scenario 3. Complete site network outage
  1. Quickly identify blast radius (some users vs all users vs remote).
  2. Check WAN/ISP link, firewall, and core switch for failures or high CPU.
  3. Inspect PoE/power, stacking & STP for loops.
  4. Bring up critical services first (internet, DNS, DHCP) and communicate.

Proactive Operations & Documentation

OPS 1. Preventing recurring incidents
  1. Perform trend analysis to find repetitive incidents and underlying causes.
  2. Convert repetitive fixes into runbooks and automation.
  3. Improve monitoring to detect earlier; tune alerts to reduce noise.
  4. Implement change controls for permanent fixes and document outcomes.
OPS 2. Documentation at scale
  1. Create standardized templates: network maps, AD layout, backup plans, escalation guides.
  2. Use a central knowledge base (Confluence/SharePoint) and version control.
  3. Update documentation in real-time during changes and post-mortems.
  4. Include runbooks and automated scripts with clear ownership and rollback steps.
Documentation = time saved
